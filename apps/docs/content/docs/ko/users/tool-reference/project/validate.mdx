## 빠른 요약
모델/텍스처/UV 규칙을 점검해 릴리즈 전 품질 이슈를 찾습니다.

## 개요
- 도구 이름: `validate`
- 카테고리: 프로젝트
- Strict schema: `true`

## 도구 설명
모델/텍스처/UV 규칙을 점검해 릴리즈 전 품질 이슈를 찾습니다.

## 입력 스키마 (JSON)
```json
{
  "type": "object",
  "additionalProperties": false,
  "properties": {
    "includeState": {
      "type": "boolean"
    },
    "ifRevision": {
      "type": "string"
    }
  }
}
```

## 필드 레퍼런스
| Field | Type | 필수 | 설명 | 제약 |
| --- | --- | --- | --- | --- |
| `includeState` | `boolean` | ❌ | - | - |
| `ifRevision` | `string` | ❌ | - | - |

## 최소 요청 예시
```json
{}
```

## 응답 예시
```json
{
  "structuredContent": {
    "findings": [
      {
        "code": "value",
        "message": "value",
        "severity": "error"
      }
    ]
  }
}
```

## 운영 노트
- 변경 도구는 `ifRevision` 사용을 권장합니다.
- 스키마에 없는 필드는 보내지 마세요 (strict schema).
- `validate`로 최종 상태를 확인한 후 `export`를 수행하세요.
